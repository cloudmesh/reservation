5. REST interfaces¶

The Server has the following routes

POST /delete/submit¶

Delete a reservation

Parameters:
  • label (String) – The label of the reservation
  • start_time (Datetime) – The time a reservation starts
  • end_time (Datetime) – The time a reservation ends
  • cm_id (String) – The cloudmesh resource ID
  • user (String) – The name of the user (username)
  • project (String) – The reservation project
  • host (String) – The host machine of the reservation
  • summary (NOT SURE WHY WE ALLO PASSING IN TEH SUMMARY SEEMS A BUG, BUT MAYBE YOU CAN EXPLAIN) – whay do we search on the summary?????
GET /delete/all¶

Uses REST framework to delete all reservations made. It goes into the function model.py and call the delete_all() function.

POST /list/submit¶

List the reservations: as param it can get any of the arguments

Parameters:
  • start_time (Datetime) – The time a reservation starts
  • end_time (Datetime) – The time a reservation ends
  • cm_id (String) – The cloudmesh resource ID
  • user (String) – The name of the user (username)
  • project (String) – The reservation project
  • host (String) – The host machine of the reservation
  • label (String) – The label of the reservation
  • summary (String) – Breaf description of the reason to make this reservation
POST /add/submit¶

submit a new a reservation: All fields are required

Parameters:
  • start_time (Datetime) – The time a reservation starts
  • end_time (Datetime) – The time a reservation ends
  • cm_id (String) – The cloudmesh resource ID
  • user (String) – The name of the user (username)
  • project (String) – The reservation project
  • host (String) – The host machine of the reservation
  • label (String) – The label of the reservation
  • summary (String) – Breaf description of the reason to make this reservation
POST /add/file¶

Add a reservation uploading a csv file

:param:uses a form to upload the reservations

GET /duration/¶

List the reservations by duration

Parameters:
  • cm_id (String) – Cloudmesh Resource ID
GET /random/¶

Randomically create reservations

GET /delete/¶

Uses REST framework to delete reservations. It calls the delete template that generates a form to delete a reservation.

GET /update/¶

Update a reservation

POST /update/¶

Update a reservation

GET /chart¶

printing the timeline from mongodb

GET /list/¶

Uses REST framework to list reservations made. It calls the list() function using the list_table() function.

GET /add/¶

Uses REST framework to add new reservations. It calls the add template that generates a form to add new reservations.

GET /static/bootstrap/(path: filename)¶

Function used internally to send static files from the static folder to the browser.

New in version 0.5.

GET /list/label/(label)¶

list the reservations searching by label

GET /list/user/(user)¶

Find the reservations searching by user

Parameters:

GET /list/id/(id)¶
